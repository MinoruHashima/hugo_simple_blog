# Diary Log Template

進捗ログ（diary_log/diary.txt）の記載ルール

## フォーマット

```
[YYYY-MM-DD]

~HH:MM
- 変更内容1
- 変更内容2
- 変更内容3

--------------------------------------------------------------------------------
```

## 記載ルール

1. 新しいエントリは区切り線の上に追加する
2. 日付は `[YYYY-MM-DD HH:MM]` 形式で記載
3. 変更内容は箇条書き（`-`）で記載
4. diary.txtと、各contextの更新はdiaryに含めない
5. 各エントリの後に区切り線 
`--------------------------------------------------------------------------------` を入れる

## 記載例

```
[2025-12-20]

~11:30
- 新機能Xを追加
- バグYを修正
- ドキュメントZを更新

~10:30
- 新機能Nを追加
- ドキュメントUを更新



--------------------------------------------------------------------------------
```
