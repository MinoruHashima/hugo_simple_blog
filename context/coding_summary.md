# コーディングサマリー

## 1. システム全体構造

### アーキテクチャ
- システムタイプ：Hugo静的サイト（外部記事キュレーション型）
- フロントエンド：HTML/CSS/JS（Hugo テンプレート）
- バックエンド：なし（静的サイト）
- データベース：なし
- ページ構成：**一覧ページのみ（詳細ページなし）**

### ディレクトリ構成（主要のみ）
```
hugo_simple_blog/
├── content/posts/  # Markdown記事（要約+外部URL）
├── layouts/        # 自作テンプレート（テーマ不使用）
│   └── index.html  # トップページ（記事一覧）
├── static/         # 静的アセット
│   └── css/style.css  # カスタムCSS
├── public/         # ビルド出力（Git管理外）
└── .github/        # CI/CD設定
    └── workflows/
        └── deploy.yml
```

### データフロー
```
Markdown（要約+外部URL） 
  ↓
Hugo Build 
  ↓
静的HTML（記事カード一覧） 
  ↓
ブラウザ表示 → 外部リンククリック → 外部サイトへ
```

## 4. コーディング規約（主要のみ）

### ファイル・コード規約
- ファイル名：kebab-case（例：`positive-news-01.md`）
- テンプレート：Go template記法
- インデント：2スペース

### デザイン規約
- **日本語フォント**: `Klee One`（Google Fonts）


### Markdown frontmatter必須項目
- `title`, `date`, `external_url`, `summary`, `tags`

---

## 5. 記事フォーマット例
```markdown
---
title: "記事タイトル"
date: 2026-02-05T00:00:00+09:00  # ← JSTを明示（+09:00）
tags: ["タグ1", "タグ2"]
external_url: "https://example.com/article"
country: "日本"
source: "ニュースソース名"
summary: "記事の要約（2-3行程度）"
---
```

**注意**: `date` には必ずタイムゾーン `+09:00` を付ける。指定しないと UTC 扱いになり、日本時間の午前中に当日の記事が表示されない問題が発生する。

## 7. 多言語対応（翻訳運用）

### URL構造
| 言語 | URL |
|------|-----|
| 日本語（デフォルト） | `/`, `/posts/post-01/` |
| 英語 | `/en/`, `/en/posts/post-01/` |
| その他（例: クロアチア語） | `/hr/`, `/hr/posts/post-02/` |

### ファイル命名規則
- 日本語: `post-XX.md`
- 英語: `post-XX.en.md`
- その他: `post-XX.{言語コード}.md`

### 翻訳運用フロー
1. 日本語で `post-XX.md` を作成
2. `scripts/translate-prompt.md` のテンプレートを Claude.ai に貼り付け
3. 記事内容を貼り付けて翻訳依頼
4. 結果を `post-XX.en.md` 等として保存

### 言語自動判別
- 初回アクセス時にブラウザ言語を判定
- 手動切り替え時は localStorage に保存
- 次回以降は保存値を優先
